# 🧬 AML Multi-Omics RAG Copilot

An offline-first, privacy-preserving Retrieval-Augmented Generation (RAG) pipeline designed for pediatric Acute Myeloid Leukemia (AML) and pharmacogenomics research.

This system serves as a localized clinical research assistant, allowing users to query specialized medical literature with high accuracy. The architecture prioritizes data containment, ensuring zero data leakage to external LLM providers, making it compliant for sensitive clinical research and bioinformatics environments.

---

## 🏗️ Architecture Overview

The system is built as a modular data pipeline, transitioning raw clinical publications into a searchable semantic knowledge base.

1. **Data Ingestion:** Automated retrieval of clinical manuscripts using `Biopython` to interface with NCBI Entrez Utilities (PubMed/MEDLINE).
2. **Data Lake:** Raw XML/JSON payloads are staged in a localized, S3-compatible object store (**MinIO**).
3. **Vectorization:** Text chunks are embedded using `sentence-transformers/all-MiniLM-L6-v2` via **Qdrant's** native `fastembed` integration.
4. **LLM Inference:** Context-aware responses are generated by **Llama-3 (8B)**, running locally via **Ollama**, and orchestrated by **LangChain**.
5. **UI Layer:** A responsive, streaming interface built with **Streamlit**.

## 🚀 Getting Started

### 1. Prerequisites

Ensure the following are installed and running on your host machine:

- [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- [Ollama](https://ollama.com/)

### 2. Setup

```bash
# Clone the repository
git clone https://github.com/alab25/aml-omics-rag.git
cd aml-omics-rag

# Create and activate virtual environment
python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### 3. Initialize Infrastructure

Starts the Qdrant vector index and MinIO object store as local Docker containers:

```bash
docker-compose up -d
```

Then pull the LLM:

```bash
ollama pull llama3
```

### 4. Ingest Data

Fetches AML-related abstracts from PubMed, stages raw payloads in MinIO, and populates the Qdrant vector index:

```bash
python src/ingestion/fetch_pubmed.py
```

### 5. Run the App

```bash
streamlit run src/retrieval/app.py --server.fileWatcherType none
```

Open [http://localhost:8501](http://localhost:8501) in your browser to start querying the knowledge base.

## 💬 Example Usage

Once the app is running, you can ask questions like:

> "What are the pharmacogenomic markers associated with FLT3-ITD mutations in pediatric AML?"

> "Summarize the mechanism of action of venetoclax in AML treatment resistance."

The system retrieves the most semantically relevant passages from the local knowledge base and synthesizes a grounded response using the locally running Llama-3 model — no data ever leaves your machine.

## ⚙️ Configuration

Copy `.env.example` to `.env` and adjust as needed:

```env
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
QDRANT_HOST=localhost
QDRANT_PORT=6333
PUBMED_EMAIL=your@email.com   # Required by NCBI Entrez
```

## 🔒 Privacy & Compliance

All inference runs locally. No patient data, query content, or retrieved documents are transmitted to external APIs. This makes the system suitable for environments governed by HIPAA, IRB, or institutional data use agreements.
